# Age of Civilization — Consolidated Fix List

**Date:** 2026-04-21
**Source:** `MECHANICS_AUDIT_2026-04-21.md` (wiring audit) + design-flaw pass + economics deep-dive across Market, Monetary, Trade/Tariff, Consumption/Supply-Chain.

This document is the single source-of-truth punch list. Anything listed here is broken, half-built, or misdesigned. Companion audit documents keep the raw analysis.

Severity legend:
- 🔴 **CRITICAL** — core mechanic broken or mathematically unreachable; game-logic consequences.
- 🟠 **HIGH** — mechanic runs but produces wrong economic / strategic outcomes; exploitable.
- 🟡 **MEDIUM** — partial / dead code / tuning issue; noticeable but not game-breaking.
- 🟢 **LOW** — housekeeping, dead symbols, stale docs.

## Part A — Wiring Gaps (from mechanics audit)

### A1. Inflation never ticks 🔴
`Inflation.cpp` (300 lines) is complete. `computeInflation()` and `applyInflationEffects()` have **zero call sites in TurnProcessor.cpp**. Price level stays at default; every downstream consumer (CurrencyTrust, fiat transition eligibility, real maintenance cost, debt erosion) reads stale zeros.
- Fix: call from `EconomySimulation::executeMonetaryPolicy()` per player.

### A2. Goody huts only in headless 🟠
`placeGoodyHuts()` called at `src/tools/HeadlessSimulation.cpp:564`. **Not called in `src/app/Application.cpp`.** Interactive games ship without goody huts.
- Fix: 2 lines in Application init + per-turn reveal check.

### A3. DataLoader dormant 🟠
`DataLoader::instance().initialize(...)` has **zero call sites**. `buildings.json`, `techs.json`, `units.json`, `leaders.json` are dead data. Only `goods.json` + `recipes.json` load (via bespoke path in `EconomySimulation.cpp:127-134`).
- Fix: call `DataLoader::instance().initialize("data")` in Application / HeadlessSimulation startup. Also resolve buildings mismatch (43 constexpr vs 32 JSON — §A6).

### A4. Bonds header-only 🟠
`include/aoc/simulation/monetary/Bonds.hpp` (~250 L) declares issuance, trading, default. **No `Bonds.cpp`** (note: design-flaw agent found a Bonds.cpp but wiring agent did not — verify which is correct; if a .cpp exists it's ~400 lines but still outside TurnProcessor). Zero turn-loop invocations either way.
- Fix: verify file state; if missing, implement or delete header. If present, wire `processBondPayments()` + `processIOUPayments()` into TurnProcessor.

### A5. Orphaned `.cpp` files 🟡
Headers without implementations:
- `BuildingCapacity.hpp`
- `ProductionEfficiency.hpp`
- Possibly `Bonds.hpp` (see A4).
- Fix: implement or delete.

### A6. Buildings 43 vs 32 mismatch 🟡
`District.hpp:122` defines 43 `BUILDING_DEFS`. `data/definitions/buildings.json` has 32. DataLoader dormant (A3) hides the mismatch. Regenerate JSON from defs or delete the 11 extra constexpr entries.

### A7. Advanced-economy modules without turn-loop calls 🟡
No invocation sites found for:
- `EconomicDepth.cpp` (77 L)
- `IndustrialRevolution.cpp` (96 L) — but `industrialRevLevel` IS read by `TechUnemployment.cpp:27`, so maybe transitively touched.
- `ColonialEconomics.cpp` — **zone logic exists but `processEconomicZones(...)` not called from TurnProcessor.** (cross-ref with C11.)
- `EconomicEspionage.cpp` (50 L).
- Fix: trace through, wire or delete.

### A8. Orphaned singletons 🟡
- `applyResourceCurseEffects()` — body is empty (C3 below); doubly orphaned.
- `updateWageDemands()` — never called.
- `processMerchantShipFuel()` — never called.

### A9. `AIInvestmentController` orphaned 🟡
603 B header, 84 L impl, zero calls from `AIController`. Either flesh out or delete.

### A10. `AIFuturesTrading` / `AICommodityHoarding` global-only 🟡
Called from TurnProcessor:647 / :651, not from `AIController::manageEconomy`. All civs behave identically on commodity markets — kills personality differentiation.

### A11. Housing cap missing from turn loop 🟡
Docs describe base 4 / +2 Granary / +4 Hospital with growth throttling. No housing check ticks in `CityGrowth.cpp`. Aqueduct building (`District.hpp:172`) defined, nothing consumes it.
- Fix: either build the throttle or drop Aqueduct + doc claim.

### A12. Robot workers (`production/Automation.hpp`) 🟡
`updateCityAutomation()` declared, unclear invocation. Per-city robot worker expansion of recipe slots may not run.
- Fix: verify + wire from per-player loop.

### A13. Tests directory empty 🟠
`tests/` exists but empty. No unit or integration tests for an 80 kLOC simulation.
- Fix: add (a) determinism smoke test (seed → identical final state), (b) turn-loop smoke (N turns no assert), (c) per-subsystem unit tests for monetary transitions, combat class matchups, victory predicates.

## Part B — Architectural / Design Flaws (non-wiring)

### B1. Alliance system architecturally incomplete 🔴
`AllianceTypes.hpp` promises 5 × 3-level progression with concrete bonuses. `DiplomacyState.cpp` implements only 3 `form*` functions (Military / Research / Economic). **Cultural and Religious alliance constructors don't exist.** AI calls only `formEconomicAlliance` (`AIController.cpp:1701`). Confederation co-win depends on multi-type allied blocs — impossible for AI to produce organically.
- Fix: implement `formCulturalAlliance` + `formReligiousAlliance`. Add a generic `formAlliance(type)` dispatch. Extend `AIController::executeDiplomacyActions` to consider all 5 types using `LeaderBehavior::allianceDesire` + hidden traits.

### B2. Yield pipeline fragmented 🟠
Science computed in `CityScience.cpp`, gold elsewhere, production in `ProductionSystem.cpp`. `DistrictAdjacency::computeAdjacencyBonus` called **only at `ProductionSystem.cpp:59`** — Campus/Commercial/Harbor/HolySite adjacency bonuses silently dropped for science, gold, faith, culture yields.
- Fix: centralise a `computeCityYields()` fold that applies adjacency uniformly. Fixing the call sites alone won't close this; needs one canonical path.

### B3. Data pipeline bifurcated 🟠
Goods + recipes load via ad-hoc path in `EconomySimulation.cpp:127-134`. Buildings/techs/units/leaders designed for `DataLoader::instance().initialize()` that's never called. Two load paths, one active, one dormant. Fixing A3 alone leaves the split-brain in place.
- Fix: single bootstrap entry point that owns all data loading. Delete the ad-hoc path.

### B4. MonetarySystem.hpp 620 lines inline 🟡
No `.cpp`. Breaks testability, profiling, rebuild times. All logic inline across every TU that includes it.
- Fix: split to `MonetarySystem.cpp` with canonical non-inline implementation.

### B5. GA genome serializes dead fields 🟢
Per project memory: `BalanceParams.integrationThreshold` / `integrationTurnsRequired` still in genome slots 7/8. Mutations on those bits are no-ops. Remove next time `BalanceTuner` is touched — GA checkpoints would break otherwise.

### B6. Religious victory threshold possibly unreachable 🟡
Requires 40% follower dominance in **every** other civ's cities. 8-religion slot cap fills fast; late founders can't compete. Hostile borders block missionary pathing.
- Fix: tune to "3-of-4 civs" or "60% majority civs", or add retrocatch mechanic (religion swap on high-pressure cities).

### B7. Era VP snowball 🟡
Every 30 turns top-3 get 3/2/1 VP. Leader compounds lead. No tech-laggard-style catch-up mechanism.
- Fix: inverse-rank VP weighting, or score-gap-normalized awards.

## Part C — Economics System Deep-Dive

Split across four layers: **Market/Price Discovery**, **Monetary/Banking**, **Trade/Tariff/Sanctions**, **Consumption/Supply-Chain**. Every item below is a design flaw, not just a wire to connect.

### Market / Price Discovery

#### C1. Prices exist but don't drive production 🔴
`Market.cpp:56-58` runs a real supply/demand formula:
```
ratio = demand / supply
newPrice = basePrice * ratio^elasticity
```
**But recipe execution at `EconomySimulation.cpp:435-444` is binary `hasAllInputs`.** Profitability never enters the choice. High-margin recipes don't prioritize over low-margin. Closed loop consumption→shortage→price→production is **open.**
- Fix: recipe selection = rank by `profitability = (outputPrice × outputQty) / (Σ inputPrice × inputQty)`. Tie-break on slot cost. This single change rebuilds the feedback loop.

#### C2. Global elasticity hardcoded at 0.5 🟠
`Market.hpp:80`. Per-good `priceElasticity` on `GoodDef` is ignored. Shortage and glut respond symmetrically regardless of good (food, luxury, strategic resource all behave identically).
- Fix: use per-good elasticity from `GoodDef`. Calibrate: essentials (wheat) 0.2–0.3 (inelastic); luxuries 1.5–2.0 (elastic); strategic 0.8–1.2.

#### C3. Phantom supply on zero-stock 🟠
`Market.cpp:47-48` clamps supply via `max(..., 1)` to avoid division by zero. A zero-supply good prices as if one unit exists. No emergency state (black-market surge, forced substitution, autarky fallback) triggers.
- Fix: if `supply == 0 && demand > 0`, flag "stockout" state → multiply price by `maxPriceRatio` AND emit event consumable by trade-route AI + black market.

#### C4. Price smoothing 20/80 damps signal 🟡
`Market.cpp:68`: `newPrice = 0.8·new + 0.2·old` means 5+ turns for 50% adjustment. Production can't respond because recipes don't read price (C1), but once C1 is fixed the smoothing will starve the loop.
- Fix: drop smoothing OR make per-good-configurable (essentials sticky, luxuries volatile).

#### C5. Clamp ratios 0.2–5.0 too tight 🟡
`Market.hpp:82-86`. Genuine scarcity (one civ has the only iron source) can't price above 5× base — insufficient to trigger extreme responses (embargo dodging, risky expeditions).
- Fix: widen to 0.1–20×, or make clamp per-good (food tight, strategic wide).

#### C6. Market is globally flat 🟠
All cities report to one `Market`. Internal trade (`processInternalTradeForAllPlayers`) doesn't use prices — uses distance-based transport loss only (`InternalTrade.cpp:32,155-156`, `TRANSFER_FRACTION=0.5`). So a city can give surplus away internally while that same good trades globally at 4× base. No arbitrage pressure.
- Fix: local or regional markets (per-continent? per-empire?). Internal transfer should price at regional rate, foreign trade at global. Creates real arbitrage play.

#### C7. Trade routes static vs dynamic prices 🟠
Route cargo hard-coded at agreement time (`TradeRoute.hpp:26-28`). Settlement uses current price for gold flow, but quantity and composition never adapt. If steel 10× more expensive, route still hauls same 5 units.
- Fix: evaluate cargo composition each arrival using current prices; re-pick top-N profitable goods from origin surplus that destination demands.

#### C8. Monopoly pricing is income transfer, not market effect 🟠
`MonopolyPricing.cpp:93-102` computes `priceMultiplier` (1.5–3×). `applyMonopolyIncome()` pays monopolist from treasury pool, but **`Market::price()` never applies the multiplier**. Buyers pay base price; monopolist magically gets treasury deposits. `buyerPriceMultiplier()` (line 62-71) is defined, **not called at any trade settlement.**
- Fix: in `EconomySimulation::settleTradeInCoins`, multiply cargo value by monopolist's `buyerPriceMultiplier(goodId)`. Drop the separate "monopoly income" payment — it double-counts.

#### C9. Stockmarket decoupled from earnings 🟠
Investment value tied to `targetGDP / 1000.0` (`StockMarket.cpp:139`). GDP is stockpile value, not income. A wheat-hoarding civ looks rich, pays dividends from treasury. Taxation never runs inside `EconomySimulation::executeTurn` — treasury drained by maintenance only. Dividends therefore come from the same treasury pool players invested into: closed-loop money-shuffling, no production feedback.
- Fix: dividends = `invested × (target.netIncome / target.bookValue)` where `netIncome = grossProduction − maintenance − wages`. Separate investment principal from target treasury.

#### C10. Black market is subsidy not market 🟡
`BlackMarket.cpp:30-39` activates on embargo. Delivers `leakRate × 2.0` per critical good regardless of player deficit or price. Not price-responsive; not substitutive — player gets legal + illegal simultaneously without fungibility check.
- Fix: black market imports competes with legal supply; priced at `globalPrice × (1 + riskPremium)` where `riskPremium` scales with enforcement strength. Leakage capped at deficit × elasticity.

#### C11. Internal trade is frictionless autarky 🟠
No gold cost, no player control, no opportunity cost vs. foreign trade. A city with surplus iron gives it to sister city for free AND sells it abroad at global price. Tariffs bypassed by definition. Supply chains can be fully internalized, making foreign dependency optional.
- Fix: either (a) price internal transfers at regional-market rate with a small "fraternal discount" (e.g. 0.8×), or (b) cap internal transfer throughput per turn by road/rail infrastructure.

#### C12. Colonial zones: unilateral extraction, guaranteed revolt 🟠
`ColonialEconomics.cpp:28-68`. Colonizer GDP ≥ 2× host → plants zone. Host loses 30% raw goods, gets 15% value in gold, loses 2 loyalty/turn → doomed in ~25 turns. No consent gate. No garrison cost.
- Fix: (a) consent gate via diplomacy (treaty, tribute), (b) garrison cost scaling with host population, (c) make extraction selectable per-good, (d) extend lifetime by tying loyalty drain to garrison strength.

### Monetary / Banking

#### C13. Transition gates too weak 🔴
`MONETARY_TRANSITIONS` (`MonetarySystem.hpp:203-220`):
- Barter → Commodity: strength ≥ 3 (one copper coin qualifies).
- Commodity → GoldStandard: strength ≥ 20 — but 100 copper meets the bar as readily as 4 gold bars. Civ fakes gold-standard with base-metal reserves.
- GoldStandard → Fiat: **no `goldBackingRatio` check** — civ can transition with backing 0.1 and print immediately.
- Fiat → Digital: no real gate.
- Fix: per-system qualitative gates: commodity requires % of coinage paid as wages (not just reserves); goldBackingRatio ≥ 0.5 mandatory to enter Fiat; Digital gated on infra (tech + building + reserve status).

#### C14. Money conservation violated throughout 🔴
Accounting leaks:
- `CentralBank::printMoney()` adds to supply + treasury, no symmetric destruction path.
- `FiscalPolicy.cpp:64-69`: `moneySupply += spending; moneySupply -= taxRevenue`. Author comment (lines 44-46) admits double-counting risk. Government "spends on account" regardless of treasury balance.
- Bonds transfer treasury but don't adjust money supply; interest payments never tracked in a money-supply sink.
- Seigniorage: reserve-currency holder gains 0.5% × foreign GDP (`Inflation.cpp:277-297`); **no deduction from foreign treasuries**.
- `executeCurrencyReform()`: halves money supply on hyperinflation with no bookkeeping trail.
- Fix: introduce `MoneyFlow` ledger: every mutation (print, tax, spend, bond, interest, seigniorage, reform) is a signed entry with source + sink. Invariant check each turn: Σ entries = 0 plus seigniorage/print authorized actions.

#### C15. Fisher inflation uses nominal GDP growth 🔴
`Inflation.cpp:36-40` computes `dY/Y` from `(currentGDP − previousGDP) / previousGDP`. That's **nominal** growth. Subtracting nominal from `dM/M + dV/V` damps inflation instead of amplifying it: when prices already high, nominal GDP is high, formula under-reports inflation. The feedback sign can flip negative (should be positive).
- Fix: `realGrowth = nominalGrowth − inflationRate_prevTurn` (first-order approximation), or iterate to fixed point.

#### C16. Deficit spending bypasses the 10% print cap 🔴
`FiscalPolicy` auto-sets spending to 12% of GDP. `moneySupply` grows by spending-minus-taxes every turn regardless of `printMoney()` cap. Cap is advisory fiction.
- Fix: route all money-supply changes through `CentralBank::changeMoneySupply(delta)` with authoritative cap. Fiscal deficit can't exceed cap without explicit bond issuance.

#### C17. Currency trust shallow + slow 🟡
`CurrencyTrust.cpp:22-141`: 20% drift per turn toward target. Instantaneous inflation spike (e.g. 15%) decays trust gradually — player can sustain high printing for turns before consequences hit. `hasDefaulted` flag (`CurrencyCrisis.hpp:59`) is never read by trust. Track-record counter resets fully on single bad turn but rebuilds slowly. `bilateralTrust[MAX_PLAYERS]` declared, never written or read.
- Fix: (a) default history carries persistent multiplier ≤ 0.7 for 50 turns, (b) trust drift asymmetric — fast down, slow up, (c) either wire bilateral trust or delete.

#### C18. Crisis thresholds exploitable 🟠
- Hyperinflation: 3 consecutive turns > 25% inflation → 5-turn penalty → `executeCurrencyReform` halves money supply and halves debt in nominal terms. Player with large debt can **intentionally** trigger hyperinflation to wipe half their debt. Deliberate hyperinflation ends up +EV if debt > ~20% of GDP.
- Bank run: `debtToGold > 1.5 AND inflation > 8%` — civ can carry 1.5× debt relative to reserves indefinitely at 7% inflation.
- Sovereign default: triggers cooldown but doesn't haircut existing bonds. Repeated near-defaults viable.
- Fix: (a) reform should impose permanent trust floor and disable fiat transitions for N turns, (b) bank-run trigger tightened to debt/gold > 1.0, (c) sovereign default triggers haircut auction on outstanding bonds.

#### C19. Gresham's Law documented, not simulated 🟡
Header comment (`MonetarySystem.hpp:16-22`) cites Gresham. Code stores three coin reserves simultaneously; `effectiveCoinTier` picks the best each turn. No hoarding/displacement dynamic. Debasement flips a `discoveredByPartners` flag without anyone actually choosing to reject debased coins.
- Fix: track "circulating vs hoarded" per tier; debasement increases circulation share of low-purity, decreases effective tier; partners' acceptance rate drops with circulation purity.

#### C20. Forex is deterministic oracle 🟡
`ForexMarket.cpp:18-47 computeFundamentalRate()` is a closed-form function of GDP/interest/inflation/trust/reserve-status. Order flow dampened by 0.05, normalized by GDP — small civs can't attack large. `tradeBalance` field exists, reset each turn, never written by trade code. No speculative crisis possible.
- Fix: make rate update from cumulative order flow × half-life decay, not pure fundamentals. Wire trade balance from actual settlement flows. Allow speculative attacks by introducing a "central bank reserves threshold" below which defense fails.

#### C21. Central bank rates sluggish + asymmetric 🟡
Interest rate changes affect target velocity with 20%/turn drift → ~5 turns to bite. Debt interest accrues at nominal rate; no "real interest rate" adjustment. No automatic Taylor-rule response to inflation.
- Fix: add policy option "inflation-targeting mode" that auto-adjusts rate by Taylor rule. Compute real rate = nominal − expectedInflation; apply to bond yields.

#### C22. Seigniorage free income 🟡
Reserve currency (trust ≥ 0.8) gets 0.5% × foreign GDP added to treasury, deducted from no one. No run mechanic if trust collapses. No eligibility check that foreigners actually hold the currency.
- Fix: track "foreign reserve holdings of currency X" as state; seigniorage = holdings × interest-rate-differential × foreign-trust. Run event drains holdings if trust drops below 0.6.

### Trade / Tariff / Sanctions

#### C23. Routes permanent + uncapped 🔴
`TradeRouteSystem.cpp:271` — `trader.maxTrips = -1`. No per-civ slot limit, no tech gate, no building prereq. A civ spams routes with no ceiling. City connection gold (§C27) stacks on top with no diminishing returns.
- Fix: hard slot cap per civ scaling with monetary tier (already declared in `MonetarySystem`: 1–14 routes). Enforce at route creation; route count > cap blocks until cap grows.

#### C24. Sea/Air dominate Land without tradeoff 🟠
Sea = +50% gold × 2.5× speed; Land = 1.0× × slow. Coastal adjacency alone gates Sea; Harbor prereq was **deliberately removed** (comment `TradeRouteSystem.cpp:282-283`) because "AI rarely builds Harbor". Map geometry now directly determines economic outcome.
- Fix: restore Harbor prereq; teach AI to build Harbor before first sea route (AIBuilderController utility bump). Land routes get compensating tech unlocks (railroad → +2 speed, trucking → +25% gold).

#### C25. AI never retaliates to tariffs 🟠
`AIController.cpp:1994-2038` sets own tolls based on relation. Never reads partner's tolls. No reciprocal escalation. Trade wars one-sided.
- Fix: AI reads `diplomacy.tollRate(self, partner)` each turn; if partner toll > threshold, mirror rate + reputation penalty. Add "grievance: ExcessiveTolls" type.

#### C26. Auto-renewal ignores state change 🟠
`TradeRouteSystem.cpp:827-832` queues renewal on expiry. **No check** for: new war, new embargo, raised tolls, pillage history, partner relation collapse.
- Fix: `queueAutoRenewRequest` validates each precondition (`evaluateTradeConsent` re-run + toll-to-profit ratio check).

#### C27. City connection gold uncapped snowball 🟠
`CityConnection.cpp:58` — `CONNECTION_BONUS = 3` gold per connected city per turn, no diminishing returns, no road cost offset. Roads cost ~1 prod/tile; +3 gold/turn pays them off immediately. 10-city civ gets +30 gold passive.
- Fix: per-turn bonus decays with city count (e.g. `3 / log2(connectedCities + 1)`); or require "maintained road" (cost-per-turn), not just road-exists.

#### C28. Sanctions dodgeable via currency upgrade 🟠
`Sanctions.cpp:72-113` asset freeze only targets `copperCoinReserves`, `silverCoinReserves`, `goldBarReserves`. Fiat/Digital civs are sanctions-proof.
- Fix: freeze operates on abstract `spendableBalance` regardless of tier; fiat balances held in foreign reserves also freezable if reserve-currency tie active.

#### C29. Embargo violation toothless 🟡
Violation = −5 reputation + one grievance; route continues delivering same turn. Renewal doesn't check embargo status.
- Fix: embargo violation cancels the trip's cargo (physical interdiction), repeat violations escalate to trade-unit seizure, renewal gated by embargo absence.

#### C30. Pillage asymmetric by currency tier 🟡
CommodityMoney traders carry physical `carriedGold`; pillager steals it. Fiat/Digital traders carry nothing; settlement is electronic. Pillage penalty falls entirely on commodity civs — reverse of intent (early-era civs pillage-vulnerable, late-era immune).
- Fix: all tiers carry a "manifest value" seizable on pillage; settlement only completes on delivery. Digital civs use traceable ledger, so pillage converts only partial value (e.g. 40%) versus commodity 100%.

#### C31. Route utility scoring ignores tolls 🟡
`estimateTradeRouteIncome` (`TradeRouteSystem.cpp:1048`) returns gross gold, no toll subtraction. AI establishes routes that net negative after tolls. Science/culture spread never factored.
- Fix: estimate = `totalValue × goldMultiplier − expectedTolls(path) + tradeDiffusionValue(partnerTechGap)`.

#### C32. Merchant ship fuel orphan 🟢
`processMerchantShipFuel` declared, never called. Traders don't require ships at all — they just move faster on water.
- Fix: either require coastal cities to spawn merchant-ship unit for sea routes (with fuel consumption), or delete `NavalTrade::processMerchantShipFuel` + associated state.

### Consumption / Supply-Chain

#### C33. Shortage has no consequence 🔴
`EconomySimulation.cpp:307-315` consumption loop calls `stockpile.consumeGoods()`; return value is `[[maybe_unused]]`. On wheat shortage: nothing happens. No starvation, no growth penalty, no happiness hit, no riot, no trade urgency flag.
- Fix: propagate consumption failure → `CityGrowth` reads "shortfall ratio last turn" → growth penalty; `Happiness` reads same → −2 amenities per unmet essential; emits "shortage event" visible to UI + AI trade planner.

#### C34. Stockpiles unbounded 🟠
`ResourceComponent.hpp:24-45` — no cap, no spoilage. Civ can hoard 10⁶ wheat free of cost. Zero urgency to trade surplus; market price never reflects glut pressure beyond nominal elasticity.
- Fix: per-good cap = base + (Granary + Warehouse + Silo) × δ. Overflow spoils or forces fire-sale at 0.2× base price (which closes the glut → price-drop feedback loop).

#### C35. Dead goods in recipe graph 🟠
18 of 66 goods produced but never consumed as recipe inputs (`ProductionChain.cpp:88` validator doesn't flag orphans). Consumer Goods (105), Processed Food (70), Advanced Consumer Goods (113), coins (140-142), Robot Workers (143), Ammunition (71), Aircraft/Armored Vehicles (110-111), Telecom Equipment (112), Bronze (78), Rubber Goods (77), Interchangeable Parts (74), Construction Materials (104), Software (108).
- Fix: (a) extend `ProductionChain::build()` to flag orphan sinks, (b) wire consumption recipes (military maintenance consumes Ammunition; buildings consume Construction Materials; services consume Software) or add recipe-free consumer-demand table so population actually uses Consumer Goods / Processed Food.

#### C36. 20 raw resources tile-only 🟡
Iron Ore, Coal, Oil, Niter, Aluminum, Uranium, Cotton, Rubber, Wheat, Cattle, Wood, Stone, Sugar, Silk, Dyes, etc. No synthesis recipes. A civ without the tile can't bootstrap the good from intermediate goods. Synthetic substitutes (lab-grown, bio-oil) don't exist.
- Fix: late-tech unlocks synthesis recipes (Synthetic Oil from coal+tech, Synthetic Rubber, Vat Meat) at high energy + human-capital cost. Prevents hard lockouts when tile distribution unfavorable.

#### C37. Supply-chain health computed but not applied 🟠
`SupplyChain.cpp:17-62` tracks `supplyHealth[idx]` (0.0–1.0). `productionMultiplier()` mentioned but **never consumed by recipe execution**. Crisis only logs.
- Fix: `executeProduction` multiplies recipe output by `supplyHealth` of its inputs. Low health → low output, closes the feedback.

#### C38. Human capital literacy unused 🟠
`HumanCapital.cpp:16-65` computes `literacyRate` per player. **Zero consumers.** Education infrastructure invests into a stat that affects nothing.
- Fix: literacy multiplies science output (×(0.5 + 0.5×literacy)), reduces `TechUnemployment` displacement, unlocks late-era buildings. Currently zero payoff.

#### C39. Tech unemployment no population pressure 🟠
`TechUnemployment.cpp:17-94` computes displacement + unemployment rate. No population loss, no loyalty hit, no tax-revenue drop, no migration.
- Fix: unemployment > 15% → −1 amenity + growth halted; > 30% → pop flight to cities with lower unemployment (triggers internal migration mechanic).

#### C40. Resource curse is an empty function 🔴
`ResourceCurse.cpp` — `computeResourceCurse` computes modifiers correctly. **`applyResourceCurseEffects()` body is empty** (file:63-72). Dutch disease entirely absent; no manufacturing penalty, no currency appreciation, no happiness hit. Project memory refers to "resource curse" as a core design pillar.
- Fix: implement the function. Apply `manufacturingPenalty` to non-extraction recipes in affected player; `currencyAppreciation` into `ForexMarket` fundamental rate; `happinessPenalty` into city amenities.

#### C41. Worker-slot bottleneck silent fail 🟡
`EconomySimulation.cpp:407-410`: if `totalRecipesExecuted + recipe->workerSlots > maxSlots`, recipe skipped. No queue, no alert, no UI flag. Player doesn't know why production stalled. Slot cap = `pop/2 + robotWorkers` — no tech progression, no building bonus (Factory / Plant should lift cap).
- Fix: (a) expose "blocked recipes this turn" on city panel, (b) building-based multipliers (Factory ×1.5, Mass Production ×2.0), (c) robot workers actually increase cap.

#### C42. Fuel consumption per-city only 🟠
`EconomySimulation.cpp:167-208`. If city A runs out of coal, its plants silently fail while city B across the empire has 1000 coal sitting. No empire-level pool, internal trade doesn't prioritize fuel.
- Fix: fuel-type goods auto-route via internal trade with top priority, or introduce "empire fuel pool" reserved for powered buildings.

#### C43. Oil shocks without fallback 🟡
`EnergyDependency.cpp:18-105`. Peak oil flag set; no substitution to coal or renewables auto-applied. Shock is a tag without teeth.
- Fix: when `inOilShock`, oil-dependent recipes fall back to coal at 1.5× consumption rate (if coal available) or production × 0.5 (if not).

#### C44. Internal trade no multi-hop 🟡
`InternalTrade.cpp:87-184` moves goods city-to-city. No ore→ingot→tool chain through intermediate cities. A city with ore produces nothing if no local smelter; can't route ore through smelter-city to tool-city in one turn.
- Fix: multi-hop routing via BFS through recipe-capable cities. Or queue multi-turn transfers.

#### C45. Tools-efficiency whitelist hardcoded 🟡
`EconomySimulation.cpp:498-512`: recipes with `requiredBuilding.value ≤ 14 && ≠ 6 && ≠ 7` consume tools every 3 executions, else 60% efficiency. Hardcoded building IDs. Any reorder breaks silently. Tools good id 63 hardcoded.
- Fix: add `requiresTools: bool` + `toolConsumptionRate: int` to `BuildingDef`. Drive from data.

#### C46. Waste vs CO2 dual-tracking 🟢
`EconomySimulation.cpp:545-577` writes both `wasteAccumulated` (per-city) and `co2ContributionPerTurn` (global). No documented relationship. Appeal penalty (pollution → tile appeal) not wired.
- Fix: collapse to single pollution output per building; split at consumption side (local → appeal, global → climate).

# Part G — Money System Deep-Dive (2026-04-21)

Second pass on monetary subsystem. Root-cause logic bugs, numerical instability, accounting violations, design holes. Supplements but does not duplicate Part C13-C22 (high-level design) or Part A (wiring). All items verified against current source.

Severity: 🔴 CRITICAL = breaks invariant / corrupts state / exploitable for win. 🟠 MAJOR = wrong feedback direction / design hole. 🟡 MINOR = hysteresis / polish.

## G1. Fiat transition can enter with `moneySupply = 0` 🔴
`include/aoc/simulation/monetary/MonetarySystem.hpp:530-534`. Transition to FiatMoney unconditionally preserves `moneySupply`. If civ reaches Fiat with M=0 (via excessive `buyGold()`, sovereign default, or crisis reform), `currencyStrength()` returns 0 (line 379) but `maxTradeRoutes()` still returns 10 → state inconsistency. Downstream Fisher math assumes M > 0.
- Fix: `this->moneySupply = std::max<CurrencyAmount>(this->moneySupply, this->currencyStrength() * 2);` on Fiat entry. Same for Digital (line 536-539).

## G2. `buyGold()` / `sellGold()` can push M negative 🔴
`src/simulation/monetary/CentralBank.cpp:45, 67`.
```cpp
state.moneySupply -= totalCost;   // line 45, no floor
state.moneySupply += currencyGained;  // line 67, no overflow guard
```
Backing-ratio guard on line 48/70 checks `moneySupply > 0` — so if M flips negative, guard skips backing update and negative M persists. All Fisher / trust / crisis code reads undefined state.
- Fix: route every M change through single `adjustMoneySupply(state, delta, reason)` that enforces floor ≥ 1 and logs source. Eliminates dual-path problem.

## G3. Fisher equation uses nominal GDP growth, inverts feedback 🔴
`src/simulation/monetary/Inflation.cpp:36-40, 70`.
```cpp
gdpGrowth = (currentGDP - previousGDP) / previousGDP;  // NOMINAL
fisherInflation = moneyGrowth + velocityChange - gdpGrowth;  // wrong
```
`currentGDP` denominated in coins already includes inflation. Algebraically: `inflation = dM + dV − (realGrowth + prev_inflation)` → high inflation dampens itself via `-gdpGrowth`. Feedback sign wrong; printing runs underdetected; can enter stable-looking state at runaway M growth.
- Fix: `float realGDPGrowth = gdpGrowth - state.inflationRate;` (subtract prev-turn inflation), then `fisherInflation = moneyGrowth + velocityChange - realGDPGrowth;`. 2-line change, massive behavioral impact.

## G4. Hyperinflation reform wipes debt for free 🔴
`src/simulation/monetary/CurrencyCrisis.cpp:181-194`. `executeCurrencyReform()` halves `moneySupply` but leaves `governmentDebt` untouched → real debt burden cut 50%. Civ can deliberately hyperinflate (print to cap) to erase debt. No penalty beyond timer.
- Fix: reform applies `governmentDebt /= 2` too (real debt preserved across redenomination), AND sets `monetaryReformLockoutUntilTurn = currentTurn + 30` blocking new borrowing, AND floors `currencyTrust` to 0.3 for 50 turns. Exploit closed.

## G5. Fiscal deficit bypasses `printMoney()` cap 🔴
`src/simulation/monetary/FiscalPolicy.cpp:64-68`. `printMoney()` caps at 10% GDP/turn. Fiscal path is uncapped:
```cpp
state.moneySupply += state.governmentSpending;  // uncapped
state.moneySupply -= state.taxRevenue;
```
Dual-supply-path problem: one capped, one not. Civ with spending > tax revenue creates money at unlimited rate.
- Fix: fiscal spending should debit treasury only, not create money. Money creation belongs only in `printMoney`, `sellGold`, `monetizeDebt`. All three should route through G2 chokepoint.

## G6. Seigniorage is unilateral income, no offsetting debit 🟠
`src/simulation/monetary/Inflation.cpp:277-297`. Deepens C22. Formula is `income = Σ_foreign GDP * 0.005` — assumes all foreigners hold all reserves in reserve-currency. But `CurrencyExchangeComponent::foreignReserves` **already exists** in ForexMarket.cpp and is unused here. Money created from nothing violates Σ=0.
- Fix:
  ```cpp
  income = Σ_foreign (foreignReserves_of_me[i] * (myCBRate - theirCBRate));
  foreignReserves[i] -= income_from_i;  // paired debit
  ```
  Σ invariant holds. Currently seigniorage is magic.

## G7. Debasement bonus goes to wrong tier, violates Gresham 🟠
`src/simulation/monetary/CentralBank.cpp:106-118`. Bonus added to *highest-tier reserve held*, not *active legal tender*. Civ on Copper tier holding 3 gold bars gets bonus on gold (not in circulation per Gresham line 368). `totalCoinValue()` grows but `currencyStrength()` doesn't → accounting divergence.
- Fix: switch on `state.effectiveCoinTier` and add bonus to matching reserve pool.

## G8. Transition gate permeable via debasement 🟠
`include/aoc/simulation/monetary/MonetarySystem.hpp:209-214`. `minCurrencyStrength` reads post-debasement strength. Civ can debase 50% (max allowed) to double coin count, meet gate, transition up. Defeats "economic maturity" intent.
- Fix: gate reads `rawStrength = strength * (1 - debasement.debasementRatio)`. Or require `debasement.debasementRatio < 0.10` as prerequisite.

## G9. Fiat gate missing backing-ratio check 🟠
`include/aoc/simulation/monetary/MonetarySystem.hpp:214`. GoldStandard→Fiat requires only `minCurrencyStrength=75`. No check on `goldBackingRatio`. Civ can print until backing = 0.05, flip to Fiat painlessly — legitimizes the collapse. Real Bretton Woods abandonment cost credibility.
- Fix: require `goldBackingRatio >= 0.30` as gate, OR apply trust penalty on entry: `fiatTrust = max(0.3f, 0.7f - (0.5f - goldBackingRatio))`. Transition allowed but expensive.

## G10. Interest rate decoupled from real-rate economic effects 🟠
`src/simulation/monetary/Inflation.cpp:42-62`. Policy rate drifts velocity only (20%/turn toward target). No consumption penalty on high real rates, no savings incentive. 5→25% rate hike takes ~5 turns to bite; civ prints more in meantime. Central bank is cosmetic.
- Fix: compute `realRate = nominalRate - expectedInflation`, wire into: (a) city consumption multiplier (high real rates → lower demand), (b) bond demand (high rates → M drains to bonds), (c) investment decisions (high rates → factory builds slow).

## G11. `currencyStrength()` returns 0 for broken Fiat 🟠
`include/aoc/simulation/monetary/MonetarySystem.hpp:377-379`. If M=0 (G1/G2 paths), strength=0 but `maxTradeRoutes()` returns 10 regardless — AI and UI read inconsistent state.
- Fix: enforce M≥1 invariant globally (G2 fix covers this). Add `assert(system != FiatMoney || moneySupply > 0)` as debug guard.

## G12. Coin tier greedy promotion, no demotion hysteresis 🟡
`include/aoc/simulation/monetary/MonetarySystem.hpp:327-338`. Tier jumps to Gold as soon as `goldBarReserves >= 3`, reverts when they fall below. Temporary spikes give temporary `maxTradeRoutes` bonus — exploitable for short-burst trade.
- Fix: require tier condition sustained 3+ turns before promote/demote. Or use weighted value accumulation as criterion.

## G13. Money multiplier cap 100x has no gameplay consequence 🟡
`src/simulation/monetary/CentralBank.cpp:80-83`. `moneyMultiplier()` caps at 100x when `reserveRequirement <= 0.001`. Math guard fine, but no bank-run / credit-crunch event when reserves crash.
- Fix: when `reserveRequirement < 0.05`, trigger `LiquidityCrisisEvent` — freeze lending 3 turns, force CB intervention at trust cost. Cap 100x stays; needs consequence.

## G14. Treasury hoarding inflation penalty has wrong sign 🟡
`src/simulation/monetary/Inflation.cpp:106-109+`. Treasury-to-GDP > 2.0 adds inflation pressure. But hoarded treasury is *out of circulation* — per quantity theory it's *dis-inflationary* in gold-standard regime (M_effective = M − hoard). Sign flipped.
- Fix: treasury hoarding raises pressure only in fiat (precautionary demand). In gold-standard, it reduces effective M → deflationary. Split by system type.

## G15. Debasement cap is per-game lock, no remint path 🟡
`src/simulation/monetary/CentralBank.cpp:94-98`. Once `debasementRatio` hits 0.50 max, civ stuck at permanent trust penalty until transition out of CommodityMoney. No escape.
- Fix: add `remintCurrency(state, cost)` that decreases `debasementRatio` by 0.10 at cost of 20% treasury. Expensive but escape valve.

## Part G root-cause pattern

Three silent accounting leaks across monetary system:
1. **No M invariant** — money created/destroyed in 4+ code paths (printMoney, sellGold, buyGold, fiscal spend, fiscal tax, reform, seigniorage), no single chokepoint enforcing `ΔM = debit + credit`.
2. **Nominal ≠ real confusion** — GDP, rates, debt all mixed without deflation by `priceLevel`. Shows up in Fisher eq (G3), real-rate wiring (G10), treasury penalty (G14).
3. **No ledger for cross-civ flows** — seigniorage and trade don't paired-debit the counterparty (G6). Player-independent money creation.

Single fix for all three: introduce `MoneyLedger` with append-only entries `{src, dst, amount, reason}` and invariant `Σ = 0` per turn. Route every M change through it. Same architectural shape as C14 recommendation, now with 15 concrete targets.

## Part H root-cause pattern

Three recurring architectural smells across all 6 systems:

1. **Data defined, never read** — AllianceState (H1.1), CasusBelli types (H1.5), WonderEffect fields (H4.9), belief yields (H3.4), promotion bonuses (H2.4), reputation (H1.7), great-work slot fills (H4.6). Same pattern as Part E "instrumented not integrated": fields exist, consumers absent.
2. **Asymmetric accounting** — spy missions one-sided (H5.2), event rewards one-sided (H5.8), loyalty pressure asymmetric (H4.3), flank+counter bonus compound (H2.3), trade-agreement global (H4.7). Σ≠0 pattern from Part G G6 seigniorage generalises across systems.
3. **Cap/threshold violation** — housing cap doesn't bind (H4.1), CO2 unbounded (H6.6), GP threshold unbounded (H3.9), XP not reset on promote (H2.5), alliance cycling no cooldown (H1.6), grievance permanent (H1.10). Accumulators without ceilings or drains.

Root-root: game is a large accumulation engine with few invariant guards. Next architectural pass should introduce invariant-checking layer — debug build asserts `accumulator <= cap`, `A->B entry has B->A paired entry`, `defined field has ≥1 reader`. Would catch half of Part H at compile/startup.
